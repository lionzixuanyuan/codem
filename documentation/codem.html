Overview

Creating jobs in the Scheduler can be done via the web front end and the API. For easy creation of jobs in the API, a script is installed in {path to Scheduler}bin/codem. This script is self contained and can be copied to another location if needed. The script is basically a wrapper around curl or wget.

Options

The codem script accepts a number of options, which are detailed below.
Short Long Description
-H [HOST] --host [HOST] Specifies the url to the Scheduler, defaults to http://localhost:3000/
-i [FILE] --input [FILE] [Required] Input file to transcode
-o [FILE] --output [FILE] [Required] Output file to write to
-p [NAME] --preset [NAME] [Required] Name of the preset to use
-n [LIST] --notify [LIST] List of comma separated email address and urls to notify
-v --verbose Display verbose output
-s --silent Don't output anything at all
-h --help Display help

Examples

  • To convert /files/movie.flv to /files/movie.mp4 using a preset named h264, use: {% highlight bash %} $ codem -i /files/movie.flv -o /files/movie.mp4 -p h264 {% endhighlight %}

  • To convert /files/movie.flv to /files/movie.mp4 using a preset named h264 on a Scheduler located at http://scheduler.com/ and notify notifications@host.com and http://host.com/notifications, use: {% highlight bash %} $ codem -H http://scheduler.com -i /files/movie.flv -o /files/movie.mp4 -p h264 -n notifications@host.com,http://host.com/notifications {% endhighlight %}
